## Mandatory defaults (apply to every row)

| Field | Default | Why |
|---|---|---|
| Lead Source | `Master DB` | AIPL's tag for database-sourced leads |
| Lead Sub-Status | `Not Connected` | New lead = not yet contacted |
| Lead Status | `Prospect` | All imports start as prospects |
| Source | `IMPORT` | Marks the lead as coming from a CSV import (not a web form) |
| Record Currency | `INR` | Indian Rupees |
| Record Conversion Rate | `1` | No FX conversion |
| Country | `India` | All targets are Indian |
| Email Opt-in | `singleoptinuser` | Vtiger's standard opt-in tag |
| SMS opt-in | `singleoptinuser` | Same for SMS |
| Is closed? | `No` | Lead is open |
| Engagement Score | `0` | New lead, no engagement yet |
| Request count | `0` | No interactions yet |
| Lead Generated on | today's date, `DD/MM/YYYY` | Day of import |
| State | `Maharashtra` (or source value title-cased) | Most leads are Maharashtra-based |

## Field mapping from source list → Vtiger

If the source list has these columns, map them as follows:

| Source column | Vtiger column | Notes |
|---|---|---|
| `EnterpriseName` or `Company` or `Name` | `Company` | Trim whitespace, preserve case |
| `Address` | `Street` | Full address goes in Street; no separate Line1/Line2 in Vtiger |
| `Pincode` or `Postal Code` or `PIN` | `Postal Code` | Strip decimals if Excel stored as float |
| `District` or `City` | `City` | Title-case (e.g., `MUMBAI` → `Mumbai`) |
| `State` | `State` | Title-case |
| (none — enriched) | `First Name`, `Last Name`, `Salutation`, `Designation`, `Primary Email`, `Office Phone`, `Mobile Phone`, `Website` | Fill from enrichment; leave blank if unknown |

## City normalization (apply silently)

| Source value | Output |
|---|---|
| `MUMBAI`, `MUMBAI CITY` | `Mumbai` |
| `NAVI MUMBAI`, `PAWANE`, `BELAPUR` | `Navi Mumbai` |
| `THANE` | `Thane` |
| `PUNE` | `Pune` |
| `KALYAN`, `KALYAN EAST` | `Kalyan` |
| anything else | Title-case as-is |

## Designation mapping rule

After enrichment, the found designation should be mapped to one of AIPL's 4 target buckets when possible (use word-boundary matching, NOT substring — "Director" contains "cto" but isn't a CTO!):

| Found title contains (whole word) | Map to Vtiger Designation |
|---|---|
| `CIO`, `CTO`, `CISO`, `Chief Information Officer`, `Chief Technology Officer`, `VP IT`, `VP Technology`, `Vice President IT` | `VP IT / CISO / CTO` |
| `IT Infra`, `Infrastructure Head/Manager/Lead` | `IT Infra / Sr. IT Infra` |
| `IT Procurement`, `IT Purchase`, `Procurement Head`, `Purchase Head` | `IT Procurement / Purchase` |
| `IT Head`, `Head of IT`, `Head-IT`, `IT Manager`, `Manager IT`, `Head of Technology` | `IT Manager / IT Head` |
| Anything else (Director, MD, Founder, CEO, etc.) | Keep the actual title verbatim |
| No designation found at all | Default to `IT Manager / IT Head` (most common IT entry point) |

## Additional Details column (provenance)

This 64th column is where you stash all the verification data the team needs to trust the row. Pipe-separated, in this order:

```
ROLE FLAG: ... (only if non-IT contact) | CIN: ... | LinkedIn: ... | Source: <URL> | Confidence: High/Medium/Low | Notes: ...
```

Example:
```
CIN: L65920MH1974PLC017560 | LinkedIn: https://linkedin.com/in/munjal-savla | Source: https://www.zaubacorp.com/company/MOTILAL-OSWAL | Confidence: High | Notes: Verified current CIO via company press release
```

This column is the single most important field for the team — it's how they decide whether to call the lead.
